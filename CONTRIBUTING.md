# 贡献指南

感谢你对 StoryForge 的关注！欢迎通过 Issue 或 Pull Request 参与项目。

## 如何贡献

### 报告问题

1. 在 [Issues](https://github.com/AOC2334/StoryForge/issues) 中搜索是否已有相同问题
2. 若无，请新建 Issue，并尽量包含：
   - 问题描述与复现步骤
   - 期望行为 vs 实际行为
   - 环境信息（OS、版本等）
   - 截图或日志（如适用）

### 提交代码

1. Fork 本仓库（或直接在本仓库创建分支，若你有写权限）
2. 阅读 [docs/development.md](docs/development.md) 了解分支与提交规范
3. 创建分支并完成修改
4. 确保本地检查通过
5. 发起 Pull Request，填写 PR 模板中的说明

### 文档贡献

文档类改动可直接使用 `docs/*` 分支，无需复杂 Review 流程，但请保持用语清晰、结构一致。

## 行为准则

- 尊重他人，就事论事
- 欢迎新手提问，耐心回复
- 不接受骚扰、歧视或恶意行为

## 许可证

向本项目提交的代码与文档，均视为同意在 [MIT License](LICENSE) 下发布。

## 联系方式

如有疑问，请通过 Issue 讨论，或联系维护者（见 README）。
